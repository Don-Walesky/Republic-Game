# Features

## Canonical Feature Ordering

### Wave 0 — Foundation
- project bootstrap and CI
- configuration
- logging
- event bus
- deterministic time system
- save/load framework
- world manager shell

### Wave 1 — Executive Workspace
- first Unity scene shell
- desk-centered interactions
- phone, email, visitor, calendar, notification shells

### Wave 2 — World Simulation
- country and world data primitives
- geography, population, resources, currency, government foundations
- report generation into office-facing channels

### Wave 3 — Campaign
- candidacy, polling, factions, elections, victory and loss transitions

### Wave 4 — Government
- cabinet, advisors, proposals, legitimacy, crises, allocation decisions

### Waves 5-9
- economy
- legislature
- military
- diplomacy
- multiplayer

## Feature Quality Bar

Every feature must:
- pass the Republic Test
- expose clear events and state boundaries
- include tests and persistence implications where relevant
- preserve the office-first leadership fantasy
