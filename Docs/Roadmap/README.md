# Roadmap

## Canonical Production Waves

Republic now uses a 10-wave roadmap organized around the player experience.

| Wave | Name | Outcome |
|---|---|---|
| 0 | Foundation | The skeleton works |
| 1 | Executive Workspace | The player has an office |
| 2 | World Simulation | The world lives without the player |
| 3 | Campaign | The player can run for office |
| 4 | Government | The player can govern |
| 5 | Economy | Economic consequences are visible |
| 6 | Legislature | Power must pass through parliament |
| 7 | Military | Military force carries consequences |
| 8 | Diplomacy | International politics becomes playable |
| 9 | Multiplayer | Politics becomes personal |

## Delivery Rules

- Waves are sequential and dependency-driven.
- Each wave must end in a coherent executable or playable milestone.
- Technical architecture supports waves; it does not replace them.

## Current Status

**Wave 0 is now implemented as a testable .NET foundation** with configuration, logging, eventing, time, persistence, a world shell, and CI-ready build commands.
