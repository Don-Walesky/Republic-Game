Waves**, not versions or phases.\n\n**See [PRODUCTION_WAVES.md](../.github/PRODUCTION_WAVES.md) for the complete strategy.**\n\n```\nWAVE 0  →  WAVE 1  →  WAVE 2  →  WAVE 3  →  WAVE 4  →  WAVE 5  →  WAVE 6\n   ↓         ↓         ↓         ↓         ↓         ↓         ↓\n Skeleton   Alive    Interfaced Consequential Autonomous Multiplayer Beautiful\n```\n\n### Wave Completion Gates\n\n- **Wave 0**: \"The skeleton works\" (invisible infrastructure)\n- **Wave 1**: \"The world lives without a player\" (autonomous simulation)\n- **Wave 2**: \"The player has an office\" (player can decide and influence)\n- **Wave 3**: \"Every choice matters and echoes\" (interconnected consequences)\n- **Wave 4**: \"The world doesn't need you to survive\" (intelligent AI nations)\n- **Wave 5**: \"Politics becomes personal\" (multiplayer politics)\n- **Wave 6**: \"The world looks like leadership\" (polished presentation)\n\n---\n\n## Getting Started\n\n### New to the project?\n1. Read [The Republic Manifesto](./Gameplay%20Philosophy/MANIFESTO.md) (2 min)\n2. Review [Design Principles](./Gameplay%20Philosophy/Design%20Principles.md) (10 min)\n3. Check [Production Waves](../.github/PRODUCTION_WAVES.md) for development strategy (5 min)\n4. See [Roadmap](./Roadmap/README.md) for current progress\n5. Review [Features](./Features/README.md) for what we're building\n\n### Making a design decision?\n1. Check the [Republic Test](./Gameplay%20Philosophy/Design%20Principles.md#the-republic-test) in Design Principles\n2. Review which Wave the feature belongs in\n3. Consult relevant [Feature](./Features/README.md) documentation\n4. Refer to [UX](./UX/README.md) for interface patterns\n\n### Building a system in a Wave?\n1. Read [Design Principles](./Gameplay%20Philosophy/Design%20Principles.md) for philosophy\n2. Check [Architecture](./Architecture/README.md) for technical patterns\n3. Review [TDD](./TDD/README.md) for technical specs\n4. See [Features](./Features/README.md) for Wave-specific specs\n5. Implement per specification\n6. Move issue to \"Done\" on Wave project board\n\n### Tracking progress?\n1. Check [Roadmap](./Roadmap/README.md) for current Wave status\n2. See [Milestones](https://github.com/Don-Walesky/Republic-Game/milestones) for completion gates\n3. Review [Wave project boards](https://github.com/Don-Walesky/Republic-Game/projects) for sprint details\n4. Check [GitHub Issues](https://github.com/Don-Walesky/Republic-Game/issues) labeled with current Wave\n\n---\n\n## The North Star\n\n> **Republic is not a game about ruling people.**\n>\n> **It is a game about carrying the burden of democratic leadership.**\n\nWhen in doubt, refer to this. Every design decision flows from this truth.\n\nFor every feature, ask:\n**\"Does this make the player feel the burden, responsibility, and complexity of leadership?\"**\n\nIf yes → belongs in Republic.\nIf no → redesign or omit.\n\n---\n\n## GitHub Integration\n\nRepublic Studio lives in GitHub:\n\n- **[Issues](https://github.com/Don-Walesky/Republic-Game/issues)** → All tasks, organized by Wave\n- **[Projects](https://github.com/Don-Walesky/Republic-Game/projects)** → Sprint boards for each Wave\n- **[Milestones](https://github.com/Don-Walesky/Republic-Game/milestones)** → Wave completion gates\n- **[Discussions](https://github.com/Don-Walesky/Republic-Game/discussions)** → Design debates and community feedback\n- **[Documentation](./README.md)** → This living document\n\n---\n\n## Key Documents\n\n### Strategy\n- [Republic Manifesto](./Gameplay%20Philosophy/MANIFESTO.md) – The north star\n- [Design Principles](./Gameplay%20Philosophy/Design%20Principles.md) – 16 core principles + Republic Test\n- [Production Waves](../.github/PRODUCTION_WAVES.md) – Development strategy\n\n### Development\n- [Roadmap](./Roadmap/README.md) – Timeline by Wave\n- [Features](./Features/README.md) – What to build in each Wave\n- [Architecture](./Architecture/README.md) – How to build it\n\n### Experience\n- [UX Documentation](./UX/README.md) – Player interface design\n- [Gameplay Philosophy](./Gameplay%20Philosophy/README.md) – Design vision\n\n---\n\n## Wave-Based Workflow\n\n### For This Wave\n1. Check current Wave in [Roadmap](./Roadmap/README.md)\n2. Review systems to build in [Features](./Features/README.md)\n3. Check project board for your Wave\n4. Pick an issue and move it to \"In Progress\"\n5. Build per specifications\n6. Move to \"In Review\" when ready for testing\n7. Move to \"Done\" when complete\n8. Repeat until Wave complete\n\n### Wave Completion\n1. All issues in Wave project board moved to \"Done\"\n2. Milestone gate achieved\n3. Prepare for next Wave\n4. Document learnings\n5. Begin next Wave\n\n---\n\n## Success Criteria\n\n**Republic Studio is working when:**\n\n✅ All Wave work visible in project boards\n✅ Developers move issues as they work\n✅ Issues linked to design documentation\n✅ Velocity tracked and trending\n✅ Blockers identified and escalated\n✅ Wave completion gates achieved on schedule\n✅ Milestones marked complete\n✅ Everything lives in GitHub (one source of truth)\n\n---\n\n*This documentation is a living resource. Update it as the game evolves.*\n"
